Oslo no-0301

Setting a region tells the repeater which floods to pass on. It does not limit who you can reach. Packets carrying no region tag keep flooding by default, so nodes that have never been configured still get through, and every node here carries eu and no, so continent- and country-scope traffic crosses the whole mesh. The narrower tags exist so that a local city chat channel stops spending airtime being relayed across all of Norway.

What the numbers mean

A kommunenummer is not a postcode. Both are four digits in Norway, which makes them easy to mix up, but they are unrelated systems. A kommunenummer is the official code Kartverket and SSB use for each of Norway's 357 municipalities — Oslo is 0301, Bergen is 4601. A postnummer identifies a postal delivery area, and a single kommune contains many of them. Never derive one from the other.

The first two digits are the fylke. 4601 is kommune 01 in fylke 46 (Vestland); 0301 is in fylke 03 (Oslo). That is what makes the region tags useful: a node tagged no-4601 is inside no-46 by construction, so you can scope a flood to one kommune or widen it to the whole fylke without any lookup table, and the two can never contradict each other.

A city tag does not mean the node is in the city. It means the node is within 50 km of it — close enough to carry that city's traffic. A repeater on a ridge outside town is often exactly what makes routing inside the town work, so it should carry the city tag even though its address is somewhere else. It is also why a node can carry more than one: sitting between two cities, it relays for both.

Fylke, kommune and city sit side by side under no — they are independent overlays, not a nesting, since a city's 50 km radius crosses fylke borders. Region membership is not inherited from position in the tree, so every tag the node should relay is listed explicitly.

All these codes come from the map — place the node and they follow. They are the current numbers, in use since the county reform of January 2024.

A hilltop node briefly waits before re-transmitting; if it hears another repeater relay the same packet with better SNR, it drops its copy instead of duplicating local traffic on airtime that's better spent on long-haul packets.

Radio

coding rate

CR is the one radio parameter that may differ between nodes — receivers decode any CR. Higher = more redundancy, better survival through noise, but slower and more channel load. Well-connected repeater: consider 5. Frequency, bandwidth, and SF must match the mesh and are fixed here.

path.hash.mode

1-byte hashes collide quickly as repeater count grows; 2-byte (65 536 values) makes collisions practically impossible at this mesh's scale and makes network debugging far easier. 3-byte only matters at millions of nodes.

Identity

name

NO-0301

Convention: NO-<kommunenummer>-<place>, so a node on Årvollåsen in Oslo is NO-0301-Årvollåsen. The number comes from where you placed it on the map. Max 24 bytes if the node has a location set, 32 if not — Norwegian letters cost two bytes each, so å and ø count double. Takes effect after a reboot.

lat / lon

Sent as two commands, set lat and set lon — MeshCore has no combined location command. The position comes from where you clicked, so it is only as precise as that; replace it with a real GPS reading if the node's exact spot matters. Publishing a location is normal for a repeater and puts it on mesh maps, but it is public. It also drops the name limit from 32 bytes to 24.

owner.info

Contact, antenna, power, mount. Visible to anyone who queries the node — no personal info.

guest.password

Lets members query node status.

Regions

firmware
region default

Default scope tags all self-originated traffic (adverts, DMs without a known path) so it can travel through repeaters that cap unscoped flood. off leaves that traffic untagged — fine while few repeaters filter, but it won't cross one that caps unscoped flood. Country-level no keeps a DM between two people anywhere in Norway routing; scope down to the city only if you're sure all your traffic is local. Needs firmware 1.15+.
