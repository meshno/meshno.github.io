MeshCore node configurator

Config for Norwegian MeshCore nodes, following the conventions Denmark and the Netherlands use. It targets the EU/UK Narrow preset already running here — 869.618 MHz at 62.5 kHz. Set up a repeater with the regions it relays for, or a companion with the channels for where you are.

Region settings, and what the numbers mean

Neighbour reach. A node near a border serves both sides of it. These say how near counts: a repeater 5 km from the next fylke relays for that fylke too, and a companion that close is offered its channels. Your node always keeps the regions it actually stands in — this only ever adds.

km km km km

0 turns a level off. Kommune starts there: at this mesh size a kommune region has almost no members to filter, and its tags are the longest.

A region tells repeaters which floods to pass on. It does not limit who you can reach — every node carries eu and no, so wide traffic still crosses the whole mesh. Narrow tags stop a local channel flooding the country.

By default, nothing here cuts anyone off. Tags only add, and this node keeps passing untagged traffic. The one command that stops that, region denyf *, stays off unless you pick it under unscoped floods. Save it for once the mesh around you carries regions.

A kommunenummer is not a postnummer. Both are four digits and they are unrelated. Your node name uses the postnummer, because that is the number you know. Your region tags use the kommunenummer, because that is the one with real borders behind it. NO-1354 Skuiåsen tagged no-3201 is correct.

The first two digits are the fylke. So a node tagged no-3201 is inside no-32 by construction, and you can widen a flood from kommune to fylke without a lookup table.

fylkesnummer kommunenummer

A tag means "relays for", not "is in". A node inside the city radius of Oslo carries no-oslo even if it sits well outside the city — a repeater on a ridge is often exactly what makes routing inside town work. Sit between two cities and it carries both.

A border works the same way: 5 km from Buskerud and the node carries no-33 too. The numbers above set how close counts. Expect neighbours across water — over water is the best path a radio gets.

Landsdel, fylke, kommune and city are independent overlays under no, not a nesting, so every tag is listed in full. The codes come from the map and are current since the county reform of January 2024.

About the name and location

Convention: NO-<postnummer> <place> — a node on Skuiåsen in Bærums Verk is NO-1354 Skuiåsen. 32 bytes, or 24 if you publish a location. Norwegian letters cost two each. Takes effect after a reboot.

Type your postnummer into the position field and the node lands in the middle of that area — the only way to set it exactly. A map click pins the exact spot but has to guess the postnummer, and is right about two times in three: check the bar under the map, and type the number instead if it is wrong.

location sets where the node says it is, three ways. none is the default and publishes nothing. map uses the pin you placed, which is only accurate to about a hundred metres, so treat it as rough. gps lets the node find its own position, and set its clock from the satellites, on a build with GPS. Any published location puts the node on mesh maps and drops the name limit to 24 bytes.

owner.info is contact, antenna, power, mount. Visible to anyone who queries the node, so no personal info.

Passwords. Guest lets members query the node; admin lets them reconfigure it. The factory admin password is password, so set your own. Both appear in the config in the clear.

What these radio settings do

Tx power is the LoRa chip's output, defaulted to 20 dBm. Boards with an amplifier put out more, so check the manual, and blank the field to keep the board's own default. Staying inside your local legal limit is on you.

CR is the one radio parameter that may differ between nodes — receivers decode any CR. Higher = more redundancy, better survival through noise, but slower and more channel load. Well-connected repeater: consider 5. Frequency, bandwidth, and SF must match the mesh and are fixed here.

1-byte hashes collide quickly as repeater count grows; 2-byte (65 536 values) makes collisions practically impossible at this mesh's scale and makes network debugging far easier. 3-byte only matters at millions of nodes.

multi.acks sends a spare copy of each acknowledgement so it survives a collision. Steadier on a weak link, at the cost of a little more airtime. Off unless a link needs it.

Node profile

Sizes the random back-off before repeating a flood. Based on neighbour count with SNR above 0.

txdelay
rxdelay
How the back-off works, and how to choose

Several repeaters hear the same flood. Each waits a random moment before repeating it so they do not transmit over each other. The profile scales that window: the more repeaters around you, the wider it needs to be. It costs a little latency and nothing else — it does not change what your node repeats, or who can reach it.

rxdelay 3 is on every profile. It works from the receiving side: a packet that arrived weakly is held back briefly so a repeater that heard it clearly gets there first, and the weak copy is dropped as a duplicate.

A repeater should not move. One that travels keeps changing which nodes it can hear, so routes through it break as fast as they form. Set one up on the Companion tab instead — you still get the mesh, without adding an unstable relay to it.

Run neighbors on the node and count how many have SNR above 0. Reported: 16 neighbours with 4 above 0 is sparse; 50 with 19 above 0 is dense.

txdelay moves in steps of 0.2 and both delays cap at 2 — that is firmware behaviour, not opinion. Everything else here is local convention.

What default scope changes

Tags what this node sends on its own — for a repeater, mostly its adverts. It does not change which regions the node relays for; that is the tree on the left.

country (no) is the default, and where a repeater or companion should stay. Narrower scopes belong to individual channels — a local chat scoped to its kommune — and to special-purpose nodes.

off works today and stops working on any repeater set to drop unscoped packets.

Unscoped floods sets how much untagged traffic this repeater carries: messages from nodes with no region set. allow is today's mesh, where everything floods and nobody is cut off. cap keeps local untagged traffic but stops it crossing the country. deny drops it outright. Leave it on allow until the nodes around you carry regions; turning deny on early isolates any node that has not set regions yet.

            How these channels work, and who else is on them
            
The name is the channel. A channel starting with # derives its
               own key from its name, so everyone who types #baerum lands on the same
               one. There is no key to send anybody. The app accepts only a-z,
               0-9 and -, which is why these are written
               ostlandet and baerum.

            
Which is why they are not private. Anyone who knows the name can work out the
               key. Treat them as an open calling frequency; for something private, make a channel
               with a random secret and hand it out yourself.
